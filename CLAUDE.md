## 작업 원칙

- 코드 수정 전 무엇을 할지 보고한다.
- 나라님의 승인 후 실행한다.
- 한 번에 하나의 Phase 또는 하나의 작은 작업만 진행한다.

**Simplicity First** — 요청받은 범위만 구현한다. "나중에 필요할 것 같아서" 미리 만들지 않는다.
**Surgical Changes** — 요청과 직접 관련 없는 파일/코드는 건드리지 않는다. 불필요한 리팩터링 금지.

> 외부 원칙보다 이 CLAUDE.md의 Project Signal Personal 고유 규칙이 항상 우선한다.

## 금지 사항

- 기존 R&D 코드, index.html, 빌드 파일, 문서 원본 복사 금지
