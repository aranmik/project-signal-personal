## 0. 시인성 공통 규칙 (render.js 기준 — 모든 직업에 공통 적용)

전투 중 유닛에 "지속적으로 보이는" 채널은 한정되어 있다. 이걸 먼저 알아야 각 직업의 안 보이는 부분이 이해된다.

- **상태칩(유닛 하단 텍스트 칩)** — 현재 칩이 뜨는 상태는 다음뿐:
  `atkUp(공↑) / atkDown(공↓) / defUp(방↑) / defDown(방↓) / critUp(치↑) / critDown(치↓) / speedUp(속↑) / speedDown(속↓) / taunted(도발)`.
  → **poison · guard · mark · charge(충전) · rhythm · watchbow 재충전 상태는 칩이 없다** = 지속 표식이 안 보인다.
- **보호막(shield)** — HP바 위 보호막 오버레이로 **보인다**(수치 텍스트는 아님).
- **피해/회복 숫자** — `fx-number`로 뜬다(치명은 주황 치명 숫자).
- **행동선(action line)** — 타입별 색: 공격 빨강 / 원거리 녹색 / 회복 청록 / 교란(disrupt) 보라.
- **도발 "!"** — 도발당한 대상 머리 위 표식.
- **로그** — 한 줄 텍스트(버퍼가 짧아 빠르게 흘러감 — 순간 효과는 놓치기 쉬움).
- 점 마커(displayMarkers)는 **보스 HUD에서만** 사용. 일반 필드 유닛엔 안 뜬다(Legacy Marker Cleanup).

**가장 큰 시인성 공백 3가지:** ① poison 지속 표식 없음, ② 결속(forbidden/wall)·표식(tracker)·충전(mage)의 "연결/예약" 상태가 안 보임, ③ 순간 효과(게이지 감소, 우선순위 분기)는 로그가 흘러가면 사라짐.

## B. 최근 정렬 6직업 상세 (01A/01B/01C)

### 파수궁 / watchbow (01A)
- **현재 구현:** 후열 아군이 **실제 피해**(HP+보호막 풀 감소)를 입으면 공격자에게 즉시 반격. ATK 50%.
  파수궁 일반 행동 1회 사이 **최대 1회**(`counterReady`), 파수궁이 일반 행동하면 재충전. 게이지 미소모.
- **읽히는 요소:** 반격 시 원거리 행동선 + 피해 숫자(+"보복" 외침). 일반 공격과 형태가 비슷해 구분은 약함.
- **안 보이는 요소:** `counterReady`(충전/소진 상태)가 전혀 안 보임. 지금 반격 가능한지 알 수 없음.
- **추후 후보:** 보복 준비 상태칩(충전/소진), 반격 발동 전용 행동선/FX, "후열 감시자" 정체성 강화.

### 정화사 / purifier (01A)
- **현재 구현:** ① HP 30% 미만 아군 있으면 치유 우선(보호막 없음) → ② 없으면 디버프(poison/atkDown/speedDown/slow) 아군 정화 → ③ 정화 성공 시에만 보호막6 → ④ 둘 다 없으면 약한 회복 fallback/기본 공격.
- **읽히는 요소:** 회복 숫자, 보호막 바, "정화!" 외침(정화 분기 시).
- **안 보이는 요소:** **왜 이번 턴에 정화가 아니라 치유했는지**(우선순위 분기) 안 보임. 전체 위압/atkDown 상황에서 1명만 정화하는 한계도 안 읽힘.
- **추후 후보:** 정화 대상 강조, 정화 성공/실패 로그 강화. (보스 위압 vs 단일 정화 한계는 **밸런스** 별도.)

### 신관 / cleric (01B)
- **현재 구현:** **직접 회복 없음.** 생존 아군 중 보호막<8(낮음/위험) 대상 최대 2명에 보호막8. 모두 충분하면 기본 공격.
- **읽히는 목표:** 사제(회복)와 다른 "사전 보호/축복 서포터".
- **읽히는 요소:** 보호막 바 증가, "축복!" 외침, 로그("아군을 축복했다 — 보호막").
- **안 보이는 요소:** 2명 동시 부여인 점, "미리 막는다"는 의도(피해가 들어와야 가치가 드러남).
- **추후 후보:** 축복/보호막 부여 FX, 보호막 완충 체감 강화.

### 선봉 / vanguard (01B + Hotfix)
- **현재 구현:** 선봉 위치와 무관하게 **아군 전열 f0/f1 생존자**에게 defUp. 전열 적에게 낮은 AoE(×0.9). **회복 없음.** 전열 아군 없으면 후열 fallback 없음(생략).
- **읽히는 목표:** 전열 진군/방어/압박.
- **읽히는 요소:** 전열 적 다중 행동선, 전열 아군 "방↑" 칩, "진군!" 외침.
- **안 보이는 요소:** 어느 아군이 버프 대상인지(전열) 직관적이지 않음.
- **추후 후보:** 전열 아군 버프 대상 하이라이트, 전열 적 압박 행동선/FX.

### 덫꾼 / trapper (01C)
- **현재 구현:** 생존 적 최대 2명에 poison(비중독 우선, 중복 없음). 독 대상이 **기본 공격**을 받을 때 치명 확률 +0.25 보정(시전자 무관 횡단). **스킬 공격엔 보정 없음.**
- **읽히는 요소:** disrupt 행동선 + "중독!" 외침, poison tick 피해 숫자(매 틱).
- **안 보이는 요소:** **poison 지속 표식(상태칩) 없음** → 누가 독 깔렸는지 안 보임. 치명판(독 대상 치명↑)이 **확률**이라 체감 약함. "독 = 단순 DoT가 아니라 판을 까는 것"이라는 정체성이 안 읽힘.
- **추후 후보:** 독 표식 아이콘/상태칩, 독 대상 피격 시 치명 로그("독을 찔러 치명!"), 설명 강화.

### 바드 / bard (01C)
- **현재 구현:** 아군/적 랜덤 → 대상 랜덤 → 효과 랜덤. 아군: atkUp 또는 critUp. 적: speedDown 또는 actionGauge −25. 한쪽 대상 없으면 다른 쪽 fallback. (구 rhythm 확정치명 예약 코드는 씨앗으로 보존되나 현재 바드는 부여 안 함.)
- **읽히는 요소:** 아군 "공↑/치↑" 칩 또는 적 "속↓" 칩, "리듬&템포!" 외침, 분기별 로그(가락/박자/흐트러뜨림).
- **안 보이는 요소:** actionGauge 감소는 tempo 바 순간 하락이라 잘 안 보임. **랜덤이 의도인지 버그인지** 플레이어가 헷갈릴 수 있음.
- **추후 후보:** 효과 로그 문구 강화/효과 이름 표시, 음표·주사위형 FX, "리듬&템포" 이름이 랜덤 콘셉트와 다소 어긋남(명명 후보).

## D. 스킬 설명 / 도감 설명 불일치 점검

**플레이어 노출 텍스트 기준 결론: 직접적인 "잘못된 설명"은 없음.**
- 도감(codex)은 직업당 `code / 이름 / 성향(combatRole) / 상태태그`만 노출 — 스킬 설명문 자체를 플레이어에게 보여주지 않음.
- 전투 중 노출되는 스킬 "이름"(보복/정화/진군/중독/리듬&템포 등)은 실제 동작과 대체로 부합.

**내부 코드 주석/명명 불일치 (player-facing 아님 — 수정하지 않고 후보로만 정리):**
1. `skills.js` **forbidden** 주석: "근사: 마커. (WATCH: 실제 40% 전가)" → 실제 `applyDamage`는 **60/40 전가를 실제 구현**. 주석이 구현보다 보수적(오래됨). → 주석 정정 후보.
2. `skills.js` **wall** 주석: "근사: 최저 아군 보호막 + 마커. (WATCH: 실제 50% 분담)" → 실제 **50/50 분담 구현됨**. → 주석 정정 후보.
3. **bard** 스킬 이름 "리듬&템포" — 현재는 완전 랜덤 콘셉트라 이름이 콘셉트와 다소 어긋남. → 명명 검토 후보(예: "즉흥연주").

## E. 시인성 후보 분류 (구현하지 않음 — 후보만)

**1) 로그만 고치면 되는 것 (가장 저위험)**
- 정화사: "왜 치유/정화였는지" 분기 로그(예: "위급 아군 치유" vs "디버프 정화").
- 바드: 효과 이름 명시 로그 강화(이미 분기 로그 있음 — 표현 다듬기).
- 덫꾼: 독 대상 피격 시 치명 발동 로그("독을 찔렀다 — 치명!").
- 워든: 게이지 드레인 로그 한 줄.

**2) 상태칩이 필요한 것**
- 덫꾼 poison 지속 표식(독 칩) — 현재 poison은 칩이 없어 판이 안 읽힘(우선순위 높음).
- 파수궁 counterReady(반격 준비/소진) 칩.
- 마도 charging(충전중) 칩, 추적자 mark(표식) 칩.

**3) 행동선 / FX가 필요한 것**
- 금제 bondOffense·성벽 bondDefense의 "결속 연결선"(누가 묶였는지 보이게).
- 신관/선봉 버프 부여 대상 하이라이트.
- 바드 음표/주사위형 랜덤 연출.

**4) 밸런스 조정으로 봐야 하는 것 (시인성 아님)**
- 사자왕 위압 vs 정화사 단일 정화(전체 위압 상황에서 1명 정화의 한계).
- 덫꾼 치명 보정 수치(+0.25), 바드 게이지 감소량(25), 독 2명 — 수치 튜닝.

**5) 아직 건드리면 위험한 것**
- 결속(60/40·50/50) 재분배 경로 — 무한 연쇄 차단(dealRaw 직접 경로) 위에 FX를 얹을 때 주의.
- 2차 직업 씨앗(용창/현자/성황) — 미개방 상태 유지(개방은 별도 결정).
- crit 횡단 효과(poison→치명) — 적 poison 소스가 생기면 파티에도 적용될 수 있음(현재는 무해).

## F. 후속 작업 제안 (바로 다음 후보 3~5개)

1. **덫꾼 독 상태칩 + 치명 로그** — 독 판/치명판 정체성이 가장 안 읽힘. (상태칩 1 + 로그 1, 체감 큼)
2. **파수궁 counterReady 상태칩** — 반응형 직업의 준비 상태 가시화(소규모, 안전).
3. **정화사 분기 로그 강화** — 우선순위 판단을 로그로 노출(로그만, 최저위험).
4. **마도 충전 / 추적자 표식 상태칩** — "예약형" 직업의 상태 가시화(공통 칩 패턴 재사용).
5. **금제/성벽 결속 연결선** — 탱커 정체성(연결)의 핵심이 안 보임(난이도 중, 효과 큼).

> 권장 순서: 1 → 2 → 3 (저위험·체감 큰 로그/칩 먼저) → 이후 4 → 5(FX).
